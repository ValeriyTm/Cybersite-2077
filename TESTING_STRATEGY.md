# План тестирования

Данный документ объединяет план ручного контроля качества и описание автоматизированных сценариев (Vitest, Cypress, Storybook).

## Модуль Identity

### План ручного тестирования модуля Identity

#### 1. Регистрация без использования Google

- 1.1. Проверка обязательности заполнения всех полей.
- 1.2. Проверка невалидного ввода.
- 1.3. Работа ссылок на соглашения.
- 1.4. Работа "глаза" (Password visibility toggle).
- 1.5. Работа кнопки сброса ввода в форме.
- 1.6. С выкл. инетом в DevTools проверить появление toast с ошибкой регистрации.
- 1.7. При отправке формы на кнопке должен сработать лоадер.
- 1.8. После регистрации вывод toast с успешной регистрацией и редирект на форму логина через 1.5 сек.
- 1.9. Проверить, что в БД создался аккаунт с неподтверденным флагом (в таблице Users поле isActivated = false).

#### 2. Вход в аккаунт после регистрации, но без подтверждения

- 2.1. Попытка войти в аккаунт без подтверждения. Получить отказ и toast с неудачей.

#### 3. Работа с письмом

- 3.1. Получить письмо.
- 3.2. Нажать на ссылку - должно редиректнуть на форму логина.
- 3.3. Проверить установку флага в БД (в таблице Users поле isActivated = true).

#### 4. Вход в аккаунт после регистрации и подтверждения аккаунта

- 4.1. Проверка обязательности заполнения всех полей формы.
- 4.2. Проверка невалидного ввода.
- 4.3. Работа "глаза".
- 4.4. С выключенным интернетом в DevTools проверить появление toast с ошибкой.
- 4.5. При отправке формы на кнопке должен сработать лоадер.
- 4.6. После логина следует проверить в DevTools, что пришел access token (установится в localStorage) и refresh token (в cookie). Проверить, что в БД создалась сессия в таблице tokens.
- 4.7. Войти в аккаунт без "Remember Me" и с ним. Проверить разницу в куках (без Remember Me должна приходить сессионная кука, а с флагом - кука со сроком годности).
- 4.8. После входа в аккаунт данные для профиля должны сразу же подтягиваться (если они есть).

#### 5. Вход в аккаунт через Google

- 5.1. Проверить обе кнопки входа (на обеих формах).
- 5.2. Убедиться, что никнейм и фото автоматически подтянулись с Google при входе в аккаунт.

#### 6. Проверить кнопку выхода из аккаунта.

- 6.1 После выхода должна удаляться одна сессия конкретного пользователя в БД в таблице tokens.

#### 7. Проверить кнопку выхода со всех аккаунтов.

- 7.1 После выхода должны удаляться все сессии конкретного пользователя в БД в таблице tokens.

#### 8. Проверить кнопку удаления аккаунта.

- 8.1. Должна удаляться запись в таблице Users и сопутствующие записи в таблице tokens.
- 8.2. Должен удаляться аватар (если он был загружен) из папки apps/server/uploads.

#### 9. Проверка "Forgot Password" в форме логина.

В форме указания email:

- 9.1. Проверка невалидного ввода.
- 9.2. После отправки должен выводиться toast.

В форме ввода нового пароля:

- 9.3. Проверка обязательности заполнения всех полей.
- 9.4. Проверка невалидного ввода.
- 9.5. Работа "глаза".
- 9.6. Toast после обновления пароля.
- 9.7. Попробовать залогиниться с новым паролем.

#### 10. (В профиле) Проверка загрузки аватара

- 10.1. Загружаю аватар. Обновляю страницу и проверяю, что аватар остался.
- 10.2. Загружаю новый аватар - старый должен удалиться из папки server/uploads.

#### 11. (В профиле) Редактирование данных профиля

- 11.1. Проверка обязательности заполнения всех полей (кроме name);
- 11.2. Проверка невалидного ввода.
- 11.3. После внесения изменений они должны сразу же быть отображены в профиле. Проверить данные в БД (таблица Users).

#### 12. (В профиле) Форма смены пароля.

- 12.1. Проверка обязательности заполнения всех полей.
- 12.2. Проверка невалидного ввода.
- 12.3. Работа "глаза".
- 12.4. При отправке формы на кнопке должен сработать лоадер.
- 12.5. Появление Toast.

#### 13.Двухфакторная аутентификация для пользователей с ролью ADMIN и SUPERADMIN.

- 13.1. В БД вручную какому-нибудь пользователю добавляю роль ADMIN.
- 13.2. Под этим пользователем логинюсь. В профиле нажимаю на кнопку для активации 2FA. Откроется QR - его сканирую в приложении на телефоне, например, Aegis Authenticator и ввожу код из приложения. Должно пустить в профиль.
- 13.3. Теперь при каждом входе такого пользователя в аккаунт после ввода пароля должно появляться окно ввода кода из приложения на телефоне.

#### 14. Прочее

- 14.1. Проверить автоудаление аккаунта, который не подтвержден в течение недели. Для этого в cleanup.service.ts установить в расписании удалений не "0 0 "star" "star" "star"", а ""star" "star" "star" "star" "star"" (проверка сервисом аккаунтов будет каждую минуту), а для конкретного неподтвержденного пользователя вручную в БД поменять поле createdAt на 8 дней назад. В течение минуты должен быть удален аккаунт из БД.

#### Инструкция по запуску тестов

- Unit- и интеграционные тесты
  Выполнить из корня проекта: "cd apps/web && npm run test".
- E2E-тесты
  Выполнить из корня проекта: "cd apps/web && npx cypress run".

* Визуальное тестирование:
  Выполнить "npx chromatic --project-token=TOKEN", где TOKEN - это токен Chromatic, указанный в .env в apps/web/.env.
